# Commit convention

## I have followed the type:subject commit convention
- In this commit convention there's a type which describes the type of the commit followed by a small description describing what this specific commit does.

## Types used
- I have used various types mentioned below
    1. feat -> Describing feature. Addition or refinement of a feature.
    2. fix -> Used in case of fixing a bug or an error in a document.
    3. docs -> Used for documentation purposes.

## Why consistent message matters for team?
- Consistent commit message is very important whilst working in a team because if the commit message in  not following a consistent pattern it can become difficult for developers to understand what a specific commit does? Why was it introduced in the first place? What type of task does this commit perform, etc.
